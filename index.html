System Requirements

  • Windows 10 or later (64-bit)
  • 4 GB RAM minimum (8 GB recommended for large files)
  • 100 MB free disk space
  • Python 3.10+ (for CLI usage)

File Size Limits

DataCleaner can handle files up to 500 MB. Here's what to expect at different sizes:

File Size Status Notes
< 50 MB Optimal Full performance with complete undo history
50-100 MB Good Slightly slower operations
100-250 MB Caution Limited undo history to conserve memory
250-500 MB Warning Minimal undo history, consider CLI for better performance
> 500 MB Blocked File too large - split into smaller parts or use CLI

Importing Data

Supported File Types

Extension Description
.csv Comma-separated values
.tsv Tab-separated values
.txt Plain text (auto-detect delimiter)
.xlsx Excel 2007+ workbook

How to Import

Method 1: Drag and Drop

Simply drag a file from Windows Explorer onto the DataCleaner window.

Method 2: Browse Button

  1. Go to the Import tab
  2. Click Browse...
  3. Select your file
  4. Click Open

Method 3: Keyboard Shortcut

Press Ctrl + O to open the file dialog.

Import Options

After selecting a file, you can adjust these settings:

For CSV/TSV/TXT files:

  • Encoding - Usually auto-detected (UTF-8, Windows-1252, etc.)
  • Delimiter - Usually auto-detected (comma, semicolon, tab, pipe)
  • Has Header - Whether first row contains column names

For Excel files:

  • Sheet - Select which worksheet to import
Auto-Detection

DataCleaner automatically detects file encoding, delimiter, quote style, header rows, and column data types. Manual override is available if detection fails.

Cleaning Data

Go to the Clean tab to apply transformations to your data.

Available Transformations

Whitespace Trimming

Removes leading and trailing spaces from all text values.

Example
Before: "  John Smith  "
After:  "John Smith"

When to use: Almost always - extra spaces cause matching and sorting issues.

Empty Row Removal

Removes rows where ALL columns are empty or null.

When to use: When your data has blank rows between records.

Null Token Normalization

Converts various representations of "empty" to actual null values.

Recognized tokens include:

  • Empty string, NULL, null, Null
  • N/A, NA, n/a, na
  • None, NONE, -, --
  • #N/A, #VALUE!, undefined
  • And 40+ more variations

Header Normalization

Cleans up column names with these options:

  • Trim - Remove spaces from column names
  • Lowercase - Convert to lowercase
  • Replace Spaces - Convert spaces to underscores
  • Remove Special Characters - Remove /, \, *
  • Deduplicate Names - Add numbers to duplicate column names
Example
Before: "First Name ", "First Name", "order/date"
After:  "first_name", "first_name_2", "order_date"

Numeric Normalization

Standardizes number formats:

  • European decimals: 1.234,56 → 1234.56
  • Thousand separators: 1,000,000 → 1000000
⚠️
Ambiguous Values

Values like 1,234 (is it 1234 or 1.234?) are preserved as text unless confidence is high.

Date Normalization

Converts dates to ISO format (YYYY-MM-DD).

Supported input formats:

  • 01/02/2024 (respects day-first setting)
  • 2024-01-02
  • Jan 2, 2024
  • 02-Jan-2024

Output format: 2024-01-02

Removing Duplicates

What is a Duplicate?

A duplicate is a row that matches another row. You can define what "matches" means:

  • Exact duplicates - All columns must match
  • Key-based duplicates - Only specified columns must match
  • Fuzzy duplicates - Similar values that might be typos (see Fuzzy Matching)

Keep Strategy

When duplicates are found, which one to keep?

Strategy Keeps Use When
First First occurrence Original order matters
Last Last occurrence Latest is most current
Most Complete Row with fewest nulls Data quality varies between records

How to Deduplicate

Go to Dedupe Tab

Navigate to the Dedupe tab in the workflow panel

Select Key Columns

Choose which columns define uniqueness (e.g., email, id)

Choose Keep Strategy

Select First, Last, or Most Complete

Preview and Apply

Review the duplicates found, then click Remove Duplicates

Fuzzy Matching (Smart Duplicates)

Fuzzy matching catches near-duplicates that exact matching would miss. Perfect for catching typos and variations like:

  • "John Smith" vs "Jon Smith"
  • "123 Main St" vs "123 Main Street"
  • "ACME Corp" vs "Acme Corporation"

Similarity Algorithms

Jaro-Winkler

Best for person names. Gives extra weight to matching prefixes. Use for names like "Robert" vs "Rob".

Levenshtein

Edit distance algorithm. Catches typos and small character differences. Good general-purpose.

Soundex

Phonetic matching. Finds words that sound similar like "Smith" vs "Smyth".

Metaphone

Advanced phonetic matching. Better than Soundex for non-English names.

N-gram

Character overlap analysis. Good for addresses and company names.

Configuration Options

Option Description Default
Threshold Minimum similarity score (0.0 to 1.0) to consider a match 0.85
Match Columns Columns to compare for similarity Required
Exact Match Columns Columns that must match exactly (blocking) Optional
Case Insensitive Ignore case when comparing Yes
Ignore Titles Remove Mr, Mrs, Dr, etc. before comparing Yes
Column Weights Give certain columns more importance Equal
💡
Performance Tip

Use "Exact Match Columns" (blocking) to speed up fuzzy matching. For example, if matching customers, block on postal code - only rows with the same postal code will be compared.

Validating Data

Why Validate?

Validation helps you catch data quality issues before using your data:

  • Missing required values
  • Invalid email addresses
  • Wrong date formats
  • Unexpected values

Available Validation Rules

Rule Description Example
Required Column Column must exist in data Ensure customer_id wasn't lost
Non-Null No empty values allowed Required fields like email
Email Format Valid email structure user@domain.com
Date Valid Parseable as date Verify date columns
Numeric Valid Valid numbers only Price, quantity columns
Regex Match Custom pattern Phone: ^\d{10}$
Value in List Allowed values only Status: active, pending, closed
Unique No duplicates Verify id uniqueness

Validation Results

After running validation, you'll see:

  • Overall pass/fail status
  • Number of rows failing each rule
  • List of specific failing values
  • Option to export validation report

CSV Injection Protection

🛡️
Security Feature

Without protection, a value like =SUM(A1:A10) could execute as a formula when opened in Excel. Protection adds a single quote prefix: '=SUM(A1:A10)

SQL Export

Generate SQL INSERT statements to load your cleaned data directly into a SQL Server database.

Setting Up SQL Export

Go to SQL Export Tab

Navigate to the SQL Export tab in the workflow panel

Define Target Table

Enter the table name and schema where data will be inserted (e.g., dbo.customers)

Define Target Columns

Add each target column with its SQL data type

Map Source to Target

For each target column, select which source column maps to it

Preview and Export

Preview the generated SQL, then export to a .sql file

Supported SQL Data Types

Category Types
String VARCHAR, NVARCHAR, CHAR, NCHAR, TEXT, NTEXT
Numeric INT, BIGINT, SMALLINT, TINYINT, DECIMAL, MONEY, FLOAT, REAL
Boolean BIT
Date/Time DATE, DATETIME, DATETIME2, TIME
Other UNIQUEIDENTIFIER (GUID)

Generated SQL Features

  • Transaction Wrapping - All inserts wrapped in BEGIN/COMMIT with rollback on error
  • Batch Processing - GO statements every 1000 rows for large datasets
  • Identity Insert - Automatically handles IDENTITY columns
  • Error Handling - TRY/CATCH with detailed error messages
  • Unicode Support - N'' prefix for NVARCHAR columns
  • Null Handling - Empty values converted to SQL NULL

Example Output

SQL
-- DataCleaner SQL Export
-- Generated: 2024-01-15 14:30:00
-- Target Table: dbo.customers
-- Total Rows: 1500

SET NOCOUNT ON;
SET XACT_ABORT ON;

BEGIN TRY
    BEGIN TRANSACTION;

    INSERT INTO dbo.customers ([id], [name], [email])
    VALUES (1, N'John Smith', 'john@example.com');

    INSERT INTO dbo.customers ([id], [name], [email])
    VALUES (2, N'Jane Doe', 'jane@example.com');

    -- ... more rows ...

    COMMIT TRANSACTION;
    PRINT 'Successfully inserted 1500 rows into dbo.customers';
END TRY
BEGIN CATCH
    IF @@TRANCOUNT > 0
        ROLLBACK TRANSACTION;

    DECLARE @ErrorMessage NVARCHAR(4000) = ERROR_MESSAGE();
    RAISERROR(@ErrorMessage, 16, 1);
END CATCH;

GO

Working with Profiles

What is a Profile?

A cleaning profile saves all your settings so you can reuse them:

  • Which transformations to apply
  • Header normalization options
  • Null token list
  • Validation rules
  • Export settings

Folder Watch

Folder Watch is the "killer feature" for recurring tasks. Drop files into a watched folder and DataCleaner processes them automatically.

How It Works

  1. Configure a watch folder and output folder
  2. Select a template to apply
  3. Start watching
  4. Drop files into the watch folder
  5. Cleaned files + SQL scripts appear in output folder automatically

Watch Configuration

Setting Description
Watch Folder Folder to monitor for new files
Output Folder Where cleaned files are saved
Template Workflow template to apply
File Patterns Which files to process (e.g., *.csv, *.xlsx)
Check Interval How often to check for new files (default: 30 seconds)
Archive Originals Move processed files to archive folder

Command-Line Interface

DataCleaner includes a powerful CLI for automation and scripting. All GUI features are available via command line.

Basic Usage

bash
# Show version
datacleaner --version

# Show help
datacleaner --help

# Launch GUI explicitly
datacleaner --gui

Available Commands

Command Description
clean Clean a single file
dedupe Remove duplicates from a file
validate Validate data against rules
export Export to different formats
batch Process multiple files with glob patterns
watch Watch a folder for new files
profile Manage cleaning profiles
info Show file information

Clean Command

bash
# Using a profile
datacleaner clean data.csv --profile monthly --output cleaned.csv

# Using individual options
datacleaner clean data.csv --trim --normalize-nulls -o cleaned.csv

# Preview changes without writing
datacleaner clean data.csv --dry-run

# Output as JSON for scripting
datacleaner clean data.csv --json

Dedupe Command

bash
# By specific columns
datacleaner dedupe data.csv --columns email --keep first -o deduped.csv

# By all columns (exact duplicates)
datacleaner dedupe data.csv --keep most_complete

# Preview duplicates
datacleaner dedupe data.csv --dry-run

Validate Command

bash
# Email validation
datacleaner validate data.csv --email email_col

# Multiple rules
datacleaner validate data.csv --required name,id --unique id

# With JSON output for CI/CD
datacleaner validate data.csv --email email --json

Batch Processing

Process multiple files at once with glob patterns.

bash
# Clean all CSV files in a folder
datacleaner batch clean "data/*.csv" --profile monthly --output cleaned/

# With parallel processing (4 workers)
datacleaner batch clean "data/*.csv" --parallel 4 --output cleaned/

# Recursive search with continue on error
datacleaner batch clean "**/*.csv" --continue-on-error --output cleaned/

# Batch dedupe
datacleaner batch dedupe "data/*.csv" --columns id --output deduped/

# Batch validate with summary
datacleaner batch validate "data/*.csv" --required name,email --summary

Batch Options

Option Description
--parallel N Process N files simultaneously
--continue-on-error Don't stop if one file fails
--json Output results as JSON
--quiet Minimal output

Watch Mode

Run the folder watcher from command line.

bash
# Watch a folder
datacleaner watch ./inbox --profile auto_clean --output ./processed

# With archive of originals
datacleaner watch ./inbox --output ./processed --move-originals ./archive

# Specify file patterns
datacleaner watch ./inbox --patterns "*.csv,*.xlsx" --output ./processed

Exit Codes

Code Meaning
0 Success - all operations/validations passed
1 Error - file not found, permission denied, etc.
2 Validation failures - some rules failed
3 Batch error - some files failed

Troubleshooting

Import Issues

❌
"Encoding detection failed"

Try manually selecting an encoding (UTF-8, Windows-1252). Open in Notepad++, check encoding in bottom-right.

❌
"Delimiter not detected"

Manually specify the delimiter. Check if file has inconsistent formatting.

❌
"File too large"

Split the file into smaller parts or use the CLI for better memory handling.

Export Issues

❌
"Cannot write file"

Close the file in Excel or other applications. Check write permissions on the folder.

SQL Export Issues

❌
"Required column not mapped"

Ensure all non-nullable columns in the target table have a source column mapped.

Getting Help

  1. Check the warnings panel for details
  2. Look in the log file: %LOCALAPPDATA%\DataCleaner\logs\app.log
  3. Report issues at: GitHub Issues
