How to Import

Method 1: Drag and Drop

Simply drag a file from Windows Explorer onto the DataCleaner window.

Method 2: Browse Button

  1. Go to the Import tab
  2. Click Browse...
  3. Select your file
  4. Click Open

Method 3: Keyboard Shortcut

Import Options

After selecting a file, you can adjust these settings:

For CSV/TSV/TXT files:

  • Encoding - Usually auto-detected (UTF-8, Windows-1252, etc.)
  • Delimiter - Usually auto-detected (comma, semicolon, tab, pipe)
  • Has Header - Whether first row contains column names

For Excel files:

  • Sheet - Select which worksheet to import
Auto-Detection

DataCleaner automatically detects file encoding, delimiter, quote style, header rows, and column data types. Manual override is available if detection fails.

Strategy Behavior Use When
Union Include all columns from all files (nulls for missing) Files have slightly different columns
Intersection Only keep columns present in ALL files Want only common columns
First File Use first file's columns as template First file defines the expected schema

  • Use Full Path - Show full file path instead of just filename
üí°

Enable "Add Source Column" to know which file each row came from. This is essential for debugging and auditing merged data.

Rules and Limitations

  • Minimum 2 files required for merge
  • All files should have similar structure (same or similar columns)
  • Column names are matched case-insensitively after trimming
  • Total merged size must stay under 500 MB file limit

Cleaning Data

Go to the Clean tab to apply transformations to your data.

Available Transformations

‚úÇÔ∏è Whitespace Trimming

Removes leading and trailing spaces from all text values.

Example
Before: "  John Smith  "
After:  "John Smith"

When to use: Almost always - extra spaces cause matching and sorting issues.

üóëÔ∏è Empty Row Removal

Removes rows where ALL columns are empty or null.

When to use: When your data has blank rows between records.

üîÑ Null Token Normalization

Converts various representations of "empty" to actual null values.

Recognized tokens include:

  • Empty string, NULL, null, Null
  • N/A, NA, n/a, na
  • None, NONE, -, --
  • #N/A, #VALUE!, undefined
  • And 40+ more variations

üìù Header Normalization

Cleans up column names with these options:

  • Trim - Remove spaces from column names
  • Lowercase - Convert to lowercase
  • Replace Spaces - Convert spaces to underscores
  • Remove Special Characters - Remove /, \, *
  • Deduplicate Names - Add numbers to duplicate column names
Example
Before: "First Name ", "First Name", "order/date"
After:  "first_name", "first_name_2", "order_date"

üî¢ Numeric Normalization

Standardizes number formats:

  • European decimals: 1.234,56 ‚Üí 1234.56
  • Thousand separators: 1,000,000 ‚Üí 1000000
‚ö†Ô∏è
Ambiguous Values

Values like 1,234 (is it 1234 or 1.234?) are preserved as text unless confidence is high.

üìÖ Date Normalization

Converts dates to ISO format (YYYY-MM-DD).

Supported input formats:

  • 01/02/2024 (respects day-first setting)
  • 2024-01-02
  • Jan 2, 2024
  • 02-Jan-2024

Output format: 2024-01-02

Removing Duplicates

What is a Duplicate?

A duplicate is a row that matches another row. You can define what "matches" means:

  • Exact duplicates - All columns must match
  • Key-based duplicates - Only specified columns must match
  • Fuzzy duplicates - Similar values that might be typos

Keep Strategy

When duplicates are found, which one to keep?

Strategy Keeps Use When
First First occurrence Original order matters
Last Last occurrence Latest is most current
Most Complete Row with fewest nulls Data quality varies between records

Smart/Fuzzy Matching

Catches near-duplicates like "John Smith" vs "Jon Smith" using similarity algorithms. See the Fuzzy Matching section for complete details.

Jaro-Winkler

Best for person names. Gives extra weight to matching prefixes.

Levenshtein

Edit distance. Catches typos and small differences.

Soundex/Metaphone

Phonetic matching. Finds words that sound similar.

N-gram

Character overlap. Good for general similarity.

Fuzzy Matching NEW

Go beyond exact matching to find duplicates that are similar but not identical. Perfect for catching typos, name variations, and data entry errors.

Examples of Fuzzy Duplicates

Record A Record B Type
John Smith Jon Smith Typo (missing 'h')
Robert Johnson Bob Johnson Name variation
123 Main Street 123 Main St Abbreviation
ACME Corporation Acme Corp Case + abbreviation
Dr. Jane Doe Jane Doe Title prefix

Matching Algorithms

Jaro-Winkler (Recommended for Names)

Best algorithm for person names. Gives extra weight to characters that match at the beginning of strings.

  • Best for: First names, last names, person names
  • Score range: 0.0 (completely different) to 1.0 (identical)
  • Recommended threshold: 0.85 - 0.92
Examples
"John" vs "Jon" = 0.93 (likely same person)
"John" vs "Jane" = 0.78 (different people)
"Smith" vs "Smyth" = 0.96 (likely same)

Levenshtein (Edit Distance)

Counts the minimum number of single-character edits (insertions, deletions, substitutions) to transform one string into another.

  • Best for: Catching typos, OCR errors, data entry mistakes
  • Score range: 0.0 to 1.0 (normalized)
  • Recommended threshold: 0.80 - 0.90
Examples
"kitten" vs "sitting" = 0.57 (3 edits needed)
"example" vs "exmaple" = 0.86 (1 swap)
"test" vs "test" = 1.00 (identical)

Soundex (Phonetic)

Encodes words by how they sound in English. Two words that sound alike get the same code.

  • Best for: Names that sound similar but are spelled differently
  • Score range: 0.0, 0.25, 0.50, 0.75, or 1.0
  • Recommended threshold: 0.75 - 1.0
Examples
"Smith" vs "Smyth" = 1.0 (same code: S530)
"John" vs "Jon" = 1.0 (same code: J500)
"Stephen" vs "Steven" = 1.0 (same code: S315)

Metaphone (Advanced Phonetic)

More sophisticated phonetic algorithm that handles more English pronunciation rules than Soundex.

  • Best for: Complex names, international names
  • Score range: 0.0 to 1.0
  • Recommended threshold: 0.80 - 0.95

N-gram (Character Overlap)

Breaks strings into overlapping character sequences and measures how many they share.

  • Best for: General text similarity, addresses, company names
  • Score range: 0.0 to 1.0
  • Recommended threshold: 0.60 - 0.80
Examples
"hello" bigrams: {he, el, ll, lo}
"hallo" bigrams: {ha, al, ll, lo}
Overlap: 2/6 = 0.33

Configuration Options

Option Description Default
Match Columns Which columns to compare for fuzzy similarity Required
Algorithm Which similarity algorithm to use Jaro-Winkler
Threshold Minimum similarity score to consider a match (0.0-1.0) 0.85
Exact Match Columns Columns that must match exactly (blocking) None
Case Insensitive Ignore case when comparing Yes
Ignore Titles Remove Mr, Mrs, Dr, etc. before comparing Yes

Blocking (Performance Optimization)

For large datasets, comparing every row to every other row is slow. Use Exact Match Columns to create "blocks" - only rows that match exactly on these columns are compared.

‚ö°
Performance Tip

Set "Exact Match Columns" to a field like "State" or "City". This way, "John Smith in NY" is only compared to other NY records, not all 100,000 rows.

Preview Before Removing

Click Preview Matches to see potential duplicates before removing them. This shows:

  • Row index of each potential duplicate pair
  • Similarity score (e.g., 0.92 = 92% similar)
  • Which columns contributed to the match

Rules and Limitations

  • Fuzzy matching is slower than exact matching - use blocking for large datasets
  • Always preview matches before removing to avoid false positives
  • Lower thresholds find more matches but may include false positives
  • Higher thresholds are safer but may miss some real duplicates
  • Works best when combined with exact match blocking columns

Command-Line Interface

DataCleaner includes a powerful CLI for automation and scripting.

Basic Usage

bash
# Show version
datacleaner --version

# Show help
datacleaner --help

# Launch GUI explicitly
datacleaner --gui

Clean Command

bash
# Using a profile
datacleaner clean data.csv --profile monthly --output cleaned.csv

# Using individual options
datacleaner clean data.csv --trim --normalize-nulls -o cleaned.csv

# Preview changes without writing
datacleaner clean data.csv --dry-run

# Output as JSON for scripting
datacleaner clean data.csv --json

Dedupe Command

bash
# By specific columns
datacleaner dedupe data.csv --columns email --keep first -o deduped.csv

# By all columns (exact duplicates)
datacleaner dedupe data.csv --keep most_complete

# Preview duplicates
datacleaner dedupe data.csv --dry-run

Validate Command

bash
# Email validation
datacleaner validate data.csv --email email_col

# Multiple rules
datacleaner validate data.csv --required name,id --unique id

# With JSON output for CI/CD
datacleaner validate data.csv --email email --json

Exit Codes

Code Meaning
0 Success - all operations/validations passed
1 Error - file not found, permission denied, etc.
2 Validation failures - some rules failed

Batch Processing & Automation

Process multiple files efficiently using command-line batch operations.

Batch Clean

Clean multiple files at once with glob patterns:

bash
# Clean all CSV files in a folder
datacleaner batch clean "data/*.csv" --profile monthly --output cleaned/

# Clean with parallel processing (faster for many files)
datacleaner batch clean "data/*.csv" --parallel 4 --output cleaned/

# Continue processing even if some files fail
datacleaner batch clean "data/*.csv" --continue-on-error --output cleaned/

# Clean all Excel files recursively
datacleaner batch clean "data/**/*.xlsx" --profile strict --output cleaned/

Batch Options

Option Description
--profile Cleaning profile to apply
--output Output directory for cleaned files
--parallel N Number of parallel workers (default: 1)
--continue-on-error Don't stop on individual file errors
--dry-run Show what would be processed without doing it
